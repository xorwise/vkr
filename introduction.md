# ВВЕДЕНИЕ

Развитие городской транспортной инфраструктуры является одним из ключевых направлений
совершенствования качества жизни в современных мегаполисах. Санкт-Петербург — один из
крупнейших городов России — располагает разветвлённой сетью наземного и подземного
транспорта, включающей метрополитен, автобусные, трамвайные и троллейбусные маршруты.
Эффективное управление и анализ столь сложной системы требуют формализованного
представления знаний о её структуре, взаимосвязях и объектах инфраструктуры.

Одним из современных подходов к формализации предметных знаний является построение
онтологий — явных спецификаций концептуализации предметной области, позволяющих
описывать объекты, их свойства и отношения между ними в машиночитаемом виде. Онтологии
на основе языка OWL (Web Ontology Language) широко применяются в задачах
семантического веба, интеграции данных и интеллектуальных информационных систем.
Применительно к городскому транспорту онтологический подход позволяет представить
маршруты, остановки, виды транспортных средств и объекты городской инфраструктуры в
виде связанных знаний, пригодных для логического вывода и сложных запросов.

Вместе с тем существующие инструменты работы с OWL-онтологиями (Protégé и аналоги)
ориентированы на специалистов в области инженерии знаний и не предоставляют
удобного пользовательского интерфейса для визуализации транспортных данных на
географической карте. Практические пользователи — городские службы, аналитики,
разработчики — нуждаются в доступном веб-приложении, которое бы совмещало
онтологическое представление данных с картографической визуализацией.

Таким образом, **актуальность** настоящей работы определяется необходимостью
разработки веб-приложения, обеспечивающего удобный доступ к семантическим данным
об инфраструктуре городского транспорта Санкт-Петербурга с визуализацией на
интерактивной карте посредством Yandex Maps API.

**Целью работы** является разработка веб-приложения для работы с онтологией
маршрутов городского транспорта, обеспечивающего загрузку, хранение и отображение
семантических данных на интерактивной картографической основе.

Для достижения поставленной цели необходимо решить следующие **задачи**:

1. Провести анализ предметной области городского транспорта и существующих подходов
   к её онтологическому описанию.
2. Изучить средства построения OWL-онтологий и провести обзор аналогичных систем
   управления транспортными данными.
3. Разработать и расширить OWL-онтологию маршрутов городского транспорта
   Санкт-Петербурга, включающую классы транспортных средств, маршрутов, остановок и
   объектов городской инфраструктуры.
4. Спроектировать архитектуру веб-приложения, включая серверную и клиентскую части,
   а также интерфейс взаимодействия с онтологией.
5. Реализовать серверную часть приложения на языке Go с поддержкой парсинга
   OWL-файлов и предоставления REST API.
6. Реализовать клиентскую часть приложения на основе React с интеграцией
   Yandex Maps JavaScript API для картографической визуализации маршрутов и остановок.
7. Провести тестирование разработанного приложения и оценить корректность
   отображения онтологических данных.

**Объектом разработки** является система городского пассажирского транспорта
Санкт-Петербурга, её маршрутная сеть и объекты инфраструктуры.

**Предметом разработки** является онтологическое представление маршрутов городского
транспорта и методы его визуализации в интерактивном веб-приложении.

**Практическая значимость** работы состоит в том, что разработанное приложение
может быть использовано городскими службами, исследователями и разработчиками для
анализа транспортной инфраструктуры, выявления связей между маршрутами и объектами
города, а также как основа для интеграции с другими семантическими источниками данных.

Работа выполнена с использованием следующих технологий и инструментов: язык
программирования Go для серверной части, библиотека React для клиентской части,
Yandex Maps JavaScript API для картографической визуализации, формат OWL/XML
для представления онтологии.
